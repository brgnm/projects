# Анализ отзывов о медицинских учреждениях Алматы

Проект машинного обучения для анализа текстовых отзывов(2GIS) о государственных учреждениях (ЦОН и больницах) города Алматы. Используются методы как контролируемого, так и неконтролируемого обучения для выявления проблем и определения тональности отзывов.

---

## Цель проекта

Автоматизировать анализ отзывов пользователей для:
- **Определения тональности** (позитивный, нейтральный, негативный отзыв)
- **Выявления скрытых групп проблем** через кластеризацию текстов
- **Обнаружения закономерностей** в качестве работы учреждений
- **Принятия управленческих решений** на основе данных

## 📊 Набор данных

### Источники данных
- **ЦОН** (Центры обслуживания населения)
- **Больницы Алматы** (государственные учреждения)

### Структура данных

Каждая запись содержит информацию об **учреждении** и его **отзывах**:

| Поле | Тип | Описание |
|------|-----|---------|
| `id` | int | Уникальный идентификатор учреждения |
| `name` | string | Название учреждения |
| `category` | string | Категория (ЦОН, Больница) |
| `city` | string | Город (Алматы) |
| `district` | string | Название района |
| `address` | string | Полный адрес |
| `latitude` | float | Географическая широта |
| `longitude` | float | Географическая долгота |
| `rating` | float | Общий рейтинг учреждения (1-5) |
| `review_count` | int | Количество отзывов |
| `work_hours` | string | График работы |
| `tags` | string | Теги / категории услуг |
| `review_text` | string | Текст отзыва |
| `review_rating` | int | Оценка конкретного отзыва (1-5) |
| `review_date` | string | Дата отзыва |


## 🔄 Этапы обработки

### **Этап 1️⃣: Загрузка и подготовка** 
📓 `data_preprocessing.ipynb`

- Загрузка данных из JSON файлов
- Объединение информации об учреждениях и отзывов
- Преобразование структуры данных

### **Этап 2️⃣: Очистка данных** 
📓 `data_cleaning.ipynb`

- Удаление дубликатов и пустых записей
- Заполнение пропусков (NaN) средними значениями
- Нормализация текста:
  - Приведение к нижнему регистру
  - Удаление спецсимволов и пунктуации
  - Удаление лишних пробелов

### **Этап 3️⃣: Инженеринг признаков** 

- Преобразование рейтингов в числовой формат
- Парсинг рабочего графика → подсчёт часов работы
- **Создание целевой переменной (target):**
  - **Positive** (позитивный) ← отзывы с оценкой ≥4
  - **Neutral** (нейтральный) ← отзывы с оценкой = 3
  - **Negative** (негативный) ← отзывы с оценкой ≤2
- TF-IDF векторизация текстов для NLP моделей
- Многоязычные эмбеддинги (SentenceTransformer)

### **Этап 4️⃣: Контролируемое обучение** 
📓 `supervised_with_nlp_ml_final.ipynb`

**Задача:** Предсказать тональность отзыва на основе его текста

**Используемые модели:**
- LogisticRegression (с TF-IDF)
- MultinomialNB (Наивный Байес)
- RandomForestClassifier
- CatBoost (если применимо)

**Метрики оценки:**
- Accuracy (точность)
- F1-score (гармоническое среднее)
- ROC-AUC (площадь под кривой)
- Confusion Matrix (матрица ошибок)

### **Этап 5️⃣: Неконтролируемое обучение** 
📓 `unsupervised_ml_final.ipynb`

**Задача:** Найти естественные группы (кластеры) проблем в отзывах

**Процесс:**
1. Преобразование текстов в многоязычные эмбеддинги (SentenceTransformer)
2. Снижение размерности: 384D → 10D (UMAP)
3. Кластеризация: HDBSCAN (автоматическое определение числа кластеров)

**Результат:** Выявленные группы проблем (шум, долгие очереди, грубый персонал и т.д.)

### **Этап 6️⃣: Классификация с оптимизацией** 
📓 `classification.ipynb`

- RandomForestClassifier с GridSearch подбора гиперпараметров
- Работа с дисбалансом классов (RandomUnderSampler)
- Кросс-валидация для надежной оценки
